Tidyparse Browser Demonstration

Usage Tips: • To trigger an update, modify the editor to the left. • Use the ↑/↓ keys to scroll completions, then ↵ to apply. • Modifying the tail of a valid prefix will enter suffix mode. • Press → in suffix mode to accept the selection's next token. • Invalid snippets are marked with a squiggly underline. • Unparseable fragments are colored orange and unknown terminals are colored red inside invalid snippets. • Nonterminal stubs are enclosed in angle brackets, <F>. • Use the [Tab] key to replace nonterminal stubs. • Any symbol to the left of -> is considered a nonterminal and all other symbols are considered terminals in a CFG. • --- and -> are reserved for the following scenarios: • --- is used to separate the CFG from the test cases. • -> separates the left and right sides of a production. • Tidyparse can add ε-productions, {V} -> ε {V} | {V} ε or literals {V} -> <V> for each nonterminal, V, in the CFG. • If the grammar contains no nonterminal named START, then Tidyparse will add START -> {V} for every nonterminal, V. • Press the [Esc] key to redisplay these tips.
